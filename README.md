# Conway's Game of Life для RARS (RISC-V)

Реализация клеточного автомата «Жизнь» Джона Конвея на ассемблере RISC-V для симулятора **RARS** с использованием графического Bitmap Display. Поле фиксированного размера **32x32** с тороидальной топологией (заворачивание по краям). Каждая клетка отображается одним пикселем: живые белые (`0x00FFFFFF`), мёртвые чёрные (`0x00000000`).

## Настройка Bitmap Display в RARS
1. Установите параметры в Bitmap Display:
   - **Unit Width**: `1`
   - **Unit Height**: `1`
   - **Display Width**: `32`
   - **Display Height**: `32`
   - **Base Address**: `0x10008000`

## Запуск
1. Откройте `life.s` в RARS.
2. Убедитесь, что в рабочей директории лежат примеры `glider.txt` и `blinker.txt` (их содержимое ниже).
3. Нажмите **Assemble**, затем **Run** (F5). После сборки откройте окно Bitmap Display и нажмите **Run**.

## Интерфейс программы
После запуска выводится меню:
```
Выберите режим:
1) Ввод координат с консоли
2) Загрузка из файла
> 
```

### Режим 1: ввод с консоли
1. Введите число живых клеток `K`.
2. Введите `K` строк с парами координат `x y` (0–31). Координаты вне диапазона игнорируются.

### Режим 2: загрузка из файла
1. Введите имя файла (например, `glider.txt`).
2. Формат файла:
   ```
   K
   x1 y1
   x2 y2
   ...
   ```
   Максимум — 1024 координаты. Неверный файл или отсутствие файла приводят к запросу имени заново.

После инициализации стартовое состояние рисуется на дисплее и начинается симуляция. Вымирание (нет живых клеток) завершает программу. Для бесконечных конфигураций остановите выполнение вручную в RARS.

## Задержка и скорость
Каждый шаг содержит задержку ~100 мс через системный вызов sleep (`ecall` №32). Для ускорения или замедления измените значение в вызове `delay` в цикле симуляции.

## Примеры файлов
`glider.txt` — глайдер:
```
5
1 0
2 1
0 2
1 2
2 2
```

`blinker.txt` — осциллятор «маятник»:
```
3
1 2
2 2
3 2
```

## Тест-план
1. **Консольный ввод**: ввести глайдер (K=5 по координатам выше) — должен двигаться по диагонали, заворачиваясь по краям.
2. **Загрузка из файла**: `glider.txt` — поведение аналогично п.1.
3. **Загрузка из файла**: `blinker.txt` — осциллирует с периодом 2.
4. **Пустой вход**: `K=0` или пустой файл — немедленное завершение по вымиранию.
